◇ WHAT "SAVES" CAN BREAK

almost nothing. if you typo your bio you can just edit again. the site rebuilds every save, but builds are fast (~30 seconds) and fail-loud — if a save breaks the build, the deploy errors out and the PREVIOUS version stays live.

◇ WHAT "FIRES" CAN BREAK

less than nothing. messages have a fuse — they go away on their own. worst case: people see your typo for a minute, then it vanishes.

◇ WHERE THE DATA LIVES
SITE CONTENT → content.json (on GitHub) + images on Vercel Blob
MESSAGES → Vercel Edge Config (fast · doesn't trigger a deploy)
SAVE is for content (text, images, settings) — takes ~1 minute. FIRE is for messages — takes 3-5 seconds. both are reversible.

◇ PART TEN

TROUBLESHOOTING

if something looks wrong, run through these in order:

▸ i changed something and the site didn't update
did you press SAVE (not FIRE — that's only for messages)? wait ~1 minute for the deploy. refresh the homepage hard (Cmd+Shift+R on Mac). still wrong? open Vercel dashboard → Deployments and check for a red Error on the latest one. if it errored, your changes are in GitHub but the live site is stuck on the previous deploy. tell whoever's helping you with the site.
▸ i fired a message and no one sees it
make sure the homepage isn't muted (the 🔊 button). wait ~5 seconds — the homepage polls for new messages on a 3s loop. did you pick NOW as the fuse? other fuses delay things.
▸ an image upload failed
check the file size (over 6 MB might choke — try resizing first). check the format — only JPG / PNG / WebP / GIF. if you see "Blob storage is not configured" — setup issue, not your fault. talk to whoever set up the site.
▸ i want to undo a change
same as making a change: edit it back and save. the site doesn't have an undo button, but git history does — any past version can be restored by someone with repo access.
▸ sounds don't play
browsers don't let pages play sound until you interact (click / scroll / tap). try clicking somewhere first. iOS safari sometimes needs an extra second to wake up audio. check the 🔊 speaker button in the bottom-right.
▸ the mode flip glitched mid-transition
don't worry, it'll finish. if it really feels stuck, refresh once.
▸ everything is broken
don't panic. the previous version of the site is always one rollback away. tell whoever helps you with the site. we can restore in a couple minutes.
